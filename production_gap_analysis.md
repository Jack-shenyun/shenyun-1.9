## 四、生产记录管理

> **设计说明（根据用户纠正）**：生产记录管理采用"**统一录入、多视图查询**"的设计模式。所有记录（包括清场、首件检验、材料使用、温湿度等）均在 `production_records` 表中产生；温湿度记录、材料使用记录等是在此基础上按记录类型进行筛选展示的**查询视图页面**，而非独立的数据表。

| 需求点 | 现状 | 状态 |
| :--- | :--- | :---: |
| 生产记录单（主录入表单） | `/production/records` 已实现，支持关联生产指令、批号、工序/工位、计划量/实际量/报废量 | ✅ 已有 |
| 温湿度记录（查询视图页面） | `production_records` 表中**缺少 `recordType` 类型字段**，无法区分记录种类；温湿度查询视图页面**尚未创建** | ⚠️ 部分 |
| 材料使用记录（查询视图页面） | 同上，材料使用查询视图页面**尚未创建**；且 `production_records` 表中**缺少材料用量相关字段**（如材料编号、用量、单位） | ⚠️ 部分 |
| 清场记录（查询视图页面） | 同上，清场记录查询视图页面**尚未创建** | ⚠️ 部分 |
| 首件检验记录（查询视图页面） | 同上，首件检验查询视图页面**尚未创建** | ⚠️ 部分 |
| 所有记录关联：生产指令、产品批号、规格 | 现有表已有 `productionOrderId`、`batchNo`、`workstationName` 字段，关联基础已具备 | ✅ 已有 |
| 调批时一次性调出该批次所有相关数据 | 后端路由支持按 `productionOrderId` 过滤，但**缺少按批号聚合查询所有类型记录**的接口 | ⚠️ 部分 |

### 7.2 需求对照

| OQC 检验结果关联入库申请（检验报废数量） | OQC 表（`quality_inspections`）与 `production_warehouse_entries` 表**无直接关联字段** | ❌ 缺失 |

| IQC 检验结果与采购入库联动 | IQC 页面与仓库入库页面**无自动联动**，需手动操作 | ⚠️ 部分 |
| IPQC 关联生产指令/批号 | IPQC 已有 `batchNo`、`process` 字段，但**无 `productionOrderId` 外键关联** | ⚠️ 部分 |

### 7.3 数据库 Schema 现状

`quality_inspections` 表为 IQC/IPQC/OQC 共用的通用检验表，通过 `type` 枚举区分。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `type` | `enum('IQC','IPQC','OQC')` | 检验类型 |
| `relatedDocNo` | `varchar(50)` | 关联单据号（字符串，非外键） |
| `batchNo` | `varchar(50)` | 批号 |
| `inspectedQty` | `decimal` | 检验数量 |
| `qualifiedQty` | `decimal` | 合格数量 |
| `unqualifiedQty` | `decimal` | 不合格数量 |
| `result` | `enum('qualified','unqualified','conditional')` | 检验结论 |

**主要缺失**：`productionOrderId`（外键）、`sterilizationOrderId`（外键）、`sampleQty`（留样数量）等字段未与业务单据形成强关联。

| 采购财务 | `/purchase/finance` | 702 行 | 支持付款管理、对账 |

### 10.2 需求对照

| 需求点 | 现状 | 状态 |
| :--- | :--- | :---: |
| MRP 净需求计算（毛需求 - 库存 - 在途 + 安全库存） | MRP 页面已有净需求展示 | ✅ 已有 |
| 自动生成采购申请单（来自 MRP 计算结果） | `material_requests` 表已存在，但**MRP 页面无一键生成采购申请的功能** | ⚠️ 部分 |
| 生产部门确认采购申请 → 采购部门确认 | `material_requests` 表有 `status` 流转，但**无生产部门前置确认环节**，仅有单一审批人 | ⚠️ 部分 |
| 按产品预设供应商和采购价格自动生成采购订单草稿 | `suppliers` 表**无产品-供应商价格关联表**（缺少 `supplier_products` 或 `supplier_price_list` 表） | ❌ 缺失 |
| 采购订单审批：部门审核人审批 → 总经理审批 | `purchase_orders` 表仅有单一 `approvedBy` 字段，**无多级审批流程**（缺少部门审核人和总经理两个审批节点） | ❌ 缺失 |
| 审批通过后可打印采购订单，打印后状态变"已下达" | 未实现打印触发状态变更逻辑，当前无"已下达"状态枚举值 | ❌ 缺失 |
| 模拟创建 3-5 家供应商基础数据 | 供应商表结构完整，但**无种子数据** | ❌ 缺失 |

### 10.3 数据库 Schema 现状

`purchase_orders` 表的 `status` 枚举为：`draft → approved → ordered → partial_received → received → cancelled`，**缺少"已下达（issued）"状态**。

`suppliers` 表仅记录供应商基本信息，**缺少产品-供应商价格关联表**，无法实现按预设价格自动生成采购订单草稿。

## 十一、汇总：开发优先级建议

根据以上全量分析，按业务重要性排列开发优先级如下：

| 优先级 | 功能点 | 涉及文件 |
| :---: | :--- | :--- |
| **P0** | 生产指令类型（成品/半成品/返工）及对应批号规则 | `schema.ts`、`Orders.tsx`、`routers.ts` |
| **P0** | `production_records` 表新增 `recordType` 字段（区分清场/首件/材料使用/温湿度/通用），并补充材料用量字段 | `schema.ts`、`routers.ts` |
| **P0** | 基于 `production_records` 表创建温湿度、材料使用、清场、首件检验四个查询视图页面 | 新建页面 |
| **P0** | 灭菌单补充灭菌批号字段 + 唯一性校验 + 【到货】按钮通知质量部 | `SterilizationOrder.tsx`、`schema.ts` |
| **P0** | 入库数量公式计算（灭菌后数量 - 报废 - 留样）+ 修改原因 + 通知销售 | `ProductionWarehouseEntry.tsx`、`schema.ts` |
| **P0** | OQC 检验结果与入库申请的数据关联（检验报废数量、留样数量） | `quality_inspections` 表、`production_warehouse_entries` 表 |
| **P1** | 采购订单多级审批流（部门审核人 → 总经理）+ 打印触发"已下达"状态 | `purchase/Orders.tsx`、`schema.ts` |
| **P1** | 产品-供应商价格关联表（新增 `supplier_products` 表）+ 自动生成采购订单草稿 | `schema.ts`、`purchase/Orders.tsx` |
| **P1** | MRP 一键生成采购申请单 + 生产部门前置确认流程 | `MRP.tsx`、`routers.ts` |
| **P1** | 生产流转卡按批次汇总工序数据 + BOM 单位换算 | `ProductionRoutingCard.tsx` |
| **P2** | IQC 检验结果与仓库入库自动联动 | `quality/IQC.tsx`、`warehouse/Inbound.tsx` |
| **P2** | IPQC 增加 `productionOrderId` 外键关联 | `schema.ts`、`quality/IPQC.tsx` |
| **P2** | 生产环境管理（新增页面） | 新建 |
| **P2** | 生产工序管理（新增页面） | 新建 |
| **P2** | 销售出库单模板补充医疗器械字段（注册证号、灭菌批号等） | `warehouse/Outbound.tsx` |
| **P2** | 客户确认收货环节 | `sales/Orders.tsx` |
| **P3** | 模拟创建 3-5 家供应商种子数据 | 新建种子脚本 |
