# UI与交互统一标准（v1）

本标准用于“所有板块”的统一改造，后续新增页面必须遵循。

## 0. 视觉基线锁定
- 当前已落地的首页、业务页、图标、颜色、卡片、顶部栏、侧边栏视觉风格，视为本项目正式 UI 基线。
- 后续新增页面、弹窗、列表页、详情页，必须沿用当前风格，不允许重新切换视觉方向。
- 禁止在未确认的情况下重做整套前端风格，包括但不限于：
  - 更换整体配色体系
  - 更换图标语言
  - 更换首页布局结构
  - 更换业务页整体版式
- 后续允许的调整范围仅限于：
  - 逻辑修复
  - 字段统一
  - 审批与待办联动
  - 排版细节优化
  - 局部间距与对齐修正
  - 统一状态色和组件行为
- 如确需改 UI 风格，必须先更新本标准，再执行页面改造。

## 1. 页面结构
- 顶部：页面标题 + 搜索/筛选 + 主操作按钮
- 中部：核心业务卡片（统计、列表、待办）
- 底部：辅助信息（最近使用、版权、备注）
- 首页固定为单屏，不滚动；业务页允许内容滚动

## 2. 状态颜色（统一）
- 进行中：黄色系
- 完成：绿色系
- 不合格/失败：红色系
- 禁止新增自定义状态色；必须复用统一映射

## 3. 表格规范
- 表头固定浅底色 + 字体加粗
- 支持快速搜索
- 支持分页（20/50/100）
- 状态列统一使用状态标签组件

## 4. 表单规范
- 必填字段明确标识 `*`
- 提交前统一校验并给出中文错误提示
- 上传控件统一支持：拖拽 + 本地选择
- 动作按钮统一文案：`提交审核` / `保存草稿` / `取消`

## 5. 待办规范
- 待办分组统一为：
  - 我的待办
  - 我发起的（我创建）
  - 我处理的（分配给我）
  - 抄送我的
- 每一项必须显示红色数字角标（可为 0）

## 6. 可复用组件/常量
- 状态映射：`client/src/constants/uiStandard.ts`
- 首页工作台：`client/src/pages/Dashboard.tsx`
- 新页面必须优先复用已有样式常量，不允许同类样式重复定义
