# 全链路验收第二轮结果

> 说明：本轮继续保持“只测试、不改代码”。重点从主链切到记录链、追溯链、知识库归档、UDI、实验室与采购/IQC 的真实一致性。

## 本轮范围

- 采购 -> IQC -> 入库
- 生产记录 -> 批记录 -> 质量记录
- 实验室记录 -> 批号追溯
- UDI 标签管理
- 文件归档 / 知识库
- 电子签名 / 操作日志

## 本轮主要结论

- 采购/IQC 存在“检验失败但已经入库”的真实断点
- UDI 模块在当前环境下无法闭环，底层 `udi_labels` 表不存在
- 批记录里的实验室记录存在“系统自动绑定多个批号”的风险，容易过度追溯
- 知识库自动归档目前主要集中在销售，质量/生产/UDI 基本没有看到真实自动归档证据
- 业务操作日志极少，当前库里几乎看不到销售、生产、质量、财务的真实审计轨迹
- 生产记录数量是有的，但操作人字段几乎全空，记录可追责性很弱
- 代码层与数据库层字段/表结构分叉已经不是个别现象，会直接影响状态联动与报表一致性

## 场景结果

### Case 05 采购 -> IQC -> 入库一致性

- 真实收货单：
  - `SIM50-GR-0001` 最终状态 `warehoused`
  - `SIM50-GR-0002` 最终状态 `failed`
- 真实 IQC：
  - `SIM50-IQC-0001` 为 `passed`
  - `SIM50-IQC-0002` 为 `failed`
- 发现：
  - `SIM50-GR-0002` 虽然收货单状态为 `failed`
  - 但同批次 `SIMRM20260314-02` 已存在 `purchase_in`
  - 入库单号：`SIM50-PI-0002`
- 结论：
  - 采购质量链并没有稳定拦住不合格来料

### Case 11 生产记录 -> 批记录 -> 质量记录一致性

- 批号 `2026031201`
  - 生产状态仍为 `planned`
  - 但已经存在大量生产记录、IPQC、OQC
  - 其中 OQC `OQC-2026-0313-5297` 为 `qualified`
  - 对应仓库入库申请仍为 `pending`
- 批号 `SIMPQ20260314-01`
  - 生产状态仍为 `in_progress`
  - IPQC、OQC 均为 `unqualified`
  - 但入库申请 `SIM50-PWE-0002` 已 `completed`
  - 且已产生 `production_in`
- 结论：
  - 生产、质量、仓库三方状态并不一致
  - 批记录能展示数据，但不能证明状态链已经真实闭环

### Case 12 生产记录完整性

- 当前 `production_records` 共 `15` 条
- `operatorName` 为空的记录数：`15`
- 典型批号：
  - `2026031201` 有 `13` 条生产记录
  - `SIMB20260313-01` 有 `1` 条
  - `SIMPQ20260314-01` 有 `1` 条
- 发现：
  - 记录存在，但操作人全部缺失
- 结论：
  - 生产记录“可用但不可审计”，真实追责能力不足

### Case 15 实验室记录追溯

- 当前实验室记录很少，最新仅看到：
  - `SIM50-LAB-PWC-0001`
  - `LAB-2026-0001`
- 其中基础设施类实验室记录 `SIM50-LAB-PWC-0001`
  - `formId = pw-chemical`
  - `batchBindingMode = system-auto`
  - 自动绑定了多个批号：
    - `2026030702`
    - `2026031201`
    - `B26031201`
    - `SIMBIND20260314-01` 到 `SIMBIND20260314-05`
- 结论：
  - 实验室记录确实会进入批记录追溯
  - 但当前绑定方式偏“当天活动批次全挂上”，存在过度关联风险

### Case 16 UDI 标签管理

- 路由和页面都已接入 `udi` 模块
- Drizzle schema 也定义了 `udi_labels`
- 但当前数据库实际不存在 `udi_labels` 表
- `SHOW TABLES LIKE 'udi_labels'` 结果为空
- 结论：
  - 当前环境里的 UDI 不是“有小 bug”
  - 而是底层表都没落地，属于无法正式运行

### Case 17 文件归档 / 知识库

- 当前 `documents` 表共 `17` 条记录
- 最近新增的自动归档主要是销售订单 PDF：
  - `SO-PDF-1773490923031-446a`
  - `SO-PDF-1773490921348-46b1`
  - `SO-PDF-1773490919554-89ac`
- 收款附件也能看到 1 条真实归档：
  - `RCPT-1773223379347-596a`
- 但检索质量/生产/实验室/UDI 关键字时，没有找到对应自动归档证据
- 结论：
  - “生成文件并进知识库”目前已确认在销售链成立
  - 在质量、生产、UDI 这几条链上，尚未看到真实自动归档闭环

### Case 19 电子签名 / 审计日志

- `electronic_signatures` 当前仅看到 `1` 条真实签名
- `signature_audit_log` 当前仅看到 `1` 条真实签名审计
- `operation_logs` 当前总共仅 `3` 条
  - 且都属于 `system`
  - 没看到销售、生产、质量、财务的真实业务操作轨迹
- 结论：
  - 电子签名功能不是完全没有
  - 但真实业务审计留痕远远不够，无法支撑“所有环节完整可追溯”

### Case 20 财务记录一致性补核

- 销售订单 `S20260314-01-CODX-907658`
  - 订单状态 `completed`
  - `paymentStatus = unpaid`
  - 但对应应收 `AR-S20260314-01-CODX-907658` 已 `paid`
  - 收款记录 `REC-2026-943444` 已生成
- 预付款订单 `S20260314-03-C02-917241`
  - 销售订单 `paymentStatus = unpaid`
  - 对应应收金额 `1800000.00`
  - 但 `paidAmount = 6000000.00`
  - 收款记录 `REC-2026-921419` 也是整单 `6000000.00`
- 结论：
  - 财务记录能生成
  - 但销售单、应收单、收款记录三者口径并不总一致

## 第二轮问题清单

### P0 IQC 不合格但已存在采购入库

- 现象：
  - 收货单 `SIM50-GR-0002` 状态为 `failed`
  - 但已存在 `purchase_in`：`SIM50-PI-0002`
- 风险：
  - 不合格来料进入库存，属于真实业务风险

### P0 UDI 底层表不存在

- 现象：
  - 路由和页面存在
  - 数据库却没有 `udi_labels`
- 风险：
  - UDI 无法真实运行，标签、上传、归档链无法闭环

### P1 生产/质量/仓库状态链不一致

- 现象：
  - `2026031201` 生产仍 `planned`，但质量已做，仓库未放行
  - `SIMPQ20260314-01` 质量不合格，但仓库已入库
- 风险：
  - 批记录展示和真实放行状态可能相互矛盾

### P1 生产记录缺少操作人

- 现象：
  - `15 / 15` 条生产记录 `operatorName` 为空
- 风险：
  - 生产记录无法满足审计和责任追溯

### P1 实验室记录批号绑定过宽

- 现象：
  - 基础设施实验室记录会自动挂到多个活动批号
- 风险：
  - 批记录可能出现“看似有关，实际并非该批专属”的追溯噪音

### P1 知识库自动归档覆盖面不足

- 现象：
  - 销售单 PDF 能自动归档
  - 质量/生产/实验室/UDI 暂未看到真实自动归档证据
- 风险：
  - 上线后会出现“文件生成了，但知识库里找不到”的情况

### P1 审计日志不足

- 现象：
  - `operation_logs` 总量仅 `3` 条，且都属于系统设置
- 风险：
  - 销售、生产、质量、财务的大部分实际操作缺乏统一审计轨迹

### P1 财务口径不一致

- 现象：
  - 销售单 `paymentStatus` 不回写
  - 预付款应收金额与收款金额口径不一致
- 风险：
  - 财务对账、销售看板、应收分析会互相打架

### P1 代码模型与数据库表结构持续分叉

- 现象：
  - 已在 OQC、IQC、销售、签名、生产领料等多处碰到“代码字段名”和“真实表字段名”不一致
- 风险：
  - 这类问题最容易造成“页面可用，但联动失真”

## 当前结论

### 已确认可用的部分

- 销售订单 PDF 自动归档到知识库
- 收款附件可进入知识库
- IQC、IPQC、OQC、实验室记录、批记录页面都有真实数据基础
- 部分批号链路能形成完整样本，例如 `SIMB20260313-01`

### 已确认高风险的部分

- IQC 不合格拦截采购入库
- UDI 标签管理
- 生产/质量/仓库状态一致性
- 财务口径一致性
- 业务审计留痕
- 实验室记录精确追溯

## 下一轮建议重点

下一轮继续优先验证：

1. 真实采购型产品从销售驱动到采购单、收货、IQC、库存的完整链
2. 批记录中生产/质量/财务是否都能按单号和批号反查
3. 打印模板、标签模板、实验室模板是否真的覆盖业务使用
4. 标签打印、上传、文件管理是否存在“页面可用但底层缺表/缺落档”的类似问题
