# 全链路验收第一轮结果

> 说明：本轮仅执行高价值主链场景，未改任何代码，仅做真实数据测试与结果记录。

## 本轮范围

- Case 02 预付款订单
- Case 07 库存不足自动排产
- Case 18 OQC 合格放行

## 场景结果

### Case 01 标准账期订单

- 销售订单：`480008 / S20260314-02-C01-917241`
- 结果：
  - 订单可创建
  - 可提审
  - 审批后订单状态为 `approved`
  - 未自动生成生产计划，符合“库存充足不排产”的预期
  - 自动生成应收
- 发现：
  - 自动生成的应收状态为 `overdue`
  - 对于刚创建且刚审批通过的账期订单，这个状态很可疑
- 初步判断：
  - `dueDate` 或账期展开逻辑可能存在问题

### Case 02 预付款订单

- 销售订单：`480009 / S20260314-03-C02-917241`
- 结果：
  - 订单可创建、提审、审批
  - 收款前未生成生产计划
  - 财务收款后生成了 1 条生产计划
  - 应收状态更新为 `paid`
- 符合预期：
  - 预付款订单在收款后触发生产计划，这条主链是通的
- 发现：
  - 销售订单本身 `paymentStatus` 仍为 `unpaid`
- 初步判断：
  - 财务收款 -> 应收状态更新成功
  - 财务收款 -> 销售订单收款状态回写失败

### Case 07 库存不足自动排产

- 销售订单：`480010 / S20260314-04-C07-917241`
- 结果：
  - 订单可创建、提审、审批
  - 审批后自动生成 1 条生产计划
  - 生产计划状态为 `pending`
  - 计划数量为 `9991.0000`
- 符合预期：
  - 库存不足时，账期订单审批后自动排产
- 结论：
  - 销售 -> 生产计划 自动分流主链有效

### Case 18 OQC 合格放行

本场景结合系统现有真实数据核验，不改代码，仅核对当前真实业务记录。

#### 样本 A：合格链

- 批号：`SIMB20260313-01`
- 结果：
  - 生产订单状态：`completed`
  - IPQC：`qualified`
  - OQC：`qualified`
  - 入库申请状态：`completed`
  - 已产生 `production_in`
  - 后续已有 `sales_out`
- 结论：
  - 这条链从质量到仓库再到发货，表现为闭环

#### 样本 B：疑似“合格未放行”

- 批号：`2026031201`
- 结果：
  - 存在 `qualified` 的 OQC 记录：`OQC-2026-0313-5297`
  - 对应生产入库申请：`WE-20260313-7624`
  - 入库申请状态仍为 `pending`
  - 未产生 `production_in`
- 结论：
  - 出现“质量合格，但仓库未被放行”的真实断点

#### 样本 C：疑似“不合格却已入库”

- 批号：`SIMPQ20260314-01`
- 结果：
  - IPQC：`unqualified`
  - OQC：`unqualified`
  - 生产入库申请：`SIM50-PWE-0002`
  - 入库申请状态：`completed`
  - 已产生 `production_in`
- 结论：
  - 出现“质量不合格，但仓库仍完成入库”的严重断点

## 第一轮问题清单

### P1 财务收款后销售订单 paymentStatus 未回写

- 影响环节：财务 -> 销售
- 现象：
  - 应收单已变为 `paid`
  - 收款记录已生成
  - 销售订单仍显示 `paymentStatus = unpaid`
- 风险：
  - 销售、财务、批记录之间会出现状态不一致

### P1 账期订单刚生成即进入 overdue

- 影响环节：销售 -> 财务
- 现象：
  - 新创建的账期订单自动生成应收后，状态直接是 `overdue`
- 风险：
  - 财务看板和应收账龄会失真

### P0 OQC 合格后仓库未稳定放行

- 影响环节：质量 -> 仓库
- 现象：
  - 存在 `qualified` OQC，但入库申请仍停留在 `pending`
- 风险：
  - 质量合格后仍卡仓库，业务无法继续

### P0 OQC 不合格但仍完成入库

- 影响环节：质量 -> 仓库
- 现象：
  - 存在 `unqualified` OQC，但入库申请已 `completed`
  - 且已经产生 `production_in`
- 风险：
  - 不合格品进入库存，属于严重业务风险

### P1 OQC 回写字段与表结构疑似不一致

- 影响环节：质量 -> 入库申请
- 现象：
  - 代码中使用 `sampleRetainQty`
  - 数据库表字段实际为 `sampleQty`
- 风险：
  - OQC 回写逻辑可能部分失效或行为不稳定
